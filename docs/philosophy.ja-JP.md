# IdeaMark Philosophy

## AI時代に向けた構造原理の実装

IdeaMark は、新しい哲学理論ではありません。

それは、実装です。

IdeaMark は、既存の知的伝統から着想を得ています。とりわけ、Christopher
Alexander の *Pattern Language*、そして Roland Barthes の
*anchorage*（アンカラージュ）という概念です。\
しかし、これらの理論を再解釈したり、批評したり、拡張したりすることを目的とはしていません。\
IdeaMark
は、そこに含まれる構造的洞察を、実用的かつ機械可読な設計フレームワークへと翻訳する試みです。

本書は、その立場を明確にするためのものです。

------------------------------------------------------------------------

## 1. パターン・ランゲージから構造化された再利用へ

Christopher Alexander
は、繰り返し現れる設計上の課題を、再利用可能なパターンとして記述できると提案しました。\
パターンとは硬直した規則ではなく、文脈・問題・解決を結びつける構造化された記述です。

IdeaMark
は、この「構造化された知識を再利用する」という精神を継承しています。

ただし、建築パターンとは異なり、IdeaMark
が対象とする領域は次のように広がっています。

-   インフラ\
-   政策\
-   ビジネスシステム\
-   ソフトウェアアーキテクチャ\
-   シビックデザイン\
-   AI支援による知識実践

目的は解決策を処方することではありません。\
構造化された思考過程を保存し、それを検査し、比較し、再構成し、進化させられるようにすることです。

------------------------------------------------------------------------

## 2. 構造的洞察としてのアンカラージュ

Roland Barthes
は、言語がどのようにして画像の意味を固定・誘導するかを説明するために、*anchorage*（アンカラージュ）という概念を提示しました。\
画像は本来的に多義的であり、キャプションや文脈的枠組みによって解釈が制約されます。

IdeaMark は、この洞察を構造レベルで採用しています。

IdeaMark においては：

-   Entity（実体）は意味的に中立です。\
-   意味は、関係的役割（Occurrence）を通じて生成されます。\
-   解釈は、anchorage を持つ
    Section（セクション）によって方向づけられます。

Entityは意味的に中立であるが、
実用上は解釈を誘発する圧縮表現として設計されうる。
ただし意味の確定はOccurrenceとSectionに委ねられる。

これは意味についての哲学的主張ではありません。\
それは設計上の決定です。

> 意味を内容の内部に不可視のまま埋め込むべきではない。\
> 意味は枠組みと文脈を通じて明示されるべきである。

したがって、IdeaMark における anchorage
は理論的主張ではなく、モデリング原理です。

------------------------------------------------------------------------

## 3. 哲学的所有ではなく、構造設計

IdeaMark は、基礎となる思想について独自性を主張しません。

内容と解釈の分離、\
文脈による意味生成、\
再利用可能な知識構造の形成は、\
哲学、記号論、システム理論において広く議論されてきたテーマです。

IdeaMark が提供するのは次の点です。

-   明確な三層構造（Entity → Occurrence → Section）\
-   機械可読な YAML 仕様\
-   運用可能なツール群（CLI契約、検証ルールなど）\
-   AI支援ワークフローと互換性のあるフォーマット

これは哲学的盗用ではなく、エンジニアリング上の実装です。

------------------------------------------------------------------------

## 4. なぜそれがAI時代に重要なのか

大規模言語モデル（LLM）は、文脈依存的に意味を生成します。\
孤立した断片を解釈するのではなく、プロンプトや枠組み、対話状態の中で動作します。

この環境においては：

-   文脈は不可欠であり\
-   枠組みが出力を決定し\
-   暗黙的な構造は曖昧さを生む

IdeaMark は、この枠組みを形式化します。

解釈レイヤを明示化することで、次のことが可能になります。

-   結論がどのように形成されたかを検査する\
-   異なる枠組みを比較する\
-   進化する仮説を追跡する\
-   人間の判断を損なうことなく、機械支援を可能にする

anchorage は、ここで実装可能な構造となります。

------------------------------------------------------------------------

## 5. オープンであること

IdeaMark はオープンソースとして公開されています。

特許取得を目的とせず、専有的な技術とする意図もありません。

その根底にある構造原理は、共有された知的遺産の一部です。\
IdeaMark の役割は、それらを現代の技術環境の中で運用可能にすることです。

オープンであることにより：

-   コミュニティの利用を通じて進化でき\
-   公共知を私的所有に囲い込むことを避け\
-   共通仕様と契約を通じて相互運用性を促進します

IdeaMark
は、構造化された思考のためのシビック・インフラストラクチャを目指しています。

------------------------------------------------------------------------

## 6. 実践的な立場

IdeaMark は哲学を置き換えることを目指しません。

意味、真理、解釈を再定義しようとするものでもありません。

代わりに、控えめで具体的な立場を取ります。

> もし意味が構造と枠組みに依存するのであれば、\
> その構造と枠組みは明示され、検査可能であり、機械処理可能であるべきである。

IdeaMark は、その立場の実装です。

それ以上でもなく、それ以下でもありません。

# Part 3 — Core Model

**Version:** IdeaMark Core v1.2.0  
**Status:** Planned

This part defines the conceptual model of IdeaMark before YAML syntax and validation rules are specified.

Part 3 should explain what each Core object is responsible for and what it must not be used to represent.

## Scope

Part 3 should define concepts, responsibilities, and boundaries.

It should not define YAML syntax, schema validation details, storage design, retrieval algorithms, or authoring workflows.

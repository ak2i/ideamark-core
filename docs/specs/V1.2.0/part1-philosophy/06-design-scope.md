# 6. Design Scope

**Part:** 1 — Philosophy  
**Status:** Draft Rev005  
**Type:** Informative / Scope

This section defines the philosophical scope of IdeaMark Core.

The Core is intentionally small.

This is not because the surrounding concepts are unimportant. It is because meanings, vocabularies, coordinate systems, goals, workflows, and future intellectual activities must remain open to future projections, domains, AI systems, and human practices.

IdeaMark Core defines the engineering boundary required for reusable intellectual structures.

It intentionally leaves individual circumstances outside the Core.

## 6.1 Core as a Universal Environment

The Core should be understood as a universal environment for intellectual activities.

It is similar to a set of physical laws in a universe: it defines the minimum conditions under which many different activities can exist, move, interact, be recorded, and be reused.

The Core is not itself a journey, a goal, a policy, a workflow, or a domain theory.

Projection defines the journey.

Core defines the universe in which many journeys can occur.

## 6.2 What the Core Defines

The Core defines reusable structural mechanisms that allow intellectual activities to be represented, connected to original sources, retrieved, and reconstructed.

The Core should define:

- the existence of reusable intellectual structures;
- the distinction between structure and meaning;
- the need for traceability to original sources;
- the possibility of multiple projections over the same source;
- the conceptual roles of entities, occurrences, sections, relations, perspectives, and anchorage;
- the minimum structural requirements needed for validation and interoperability.

The Core should not define the content of a domain.

It should define the conditions under which many domains can represent and reuse intellectual activities.

## 6.3 What the Core Intentionally Does Not Define

The Core intentionally does not define:

- meaning;
- universal ontologies;
- domain vocabularies;
- universal coordinate systems;
- universal goals;
- fixed interpretations;
- cognitive models;
- optimal projections;
- authoring workflows;
- retrieval algorithms;
- storage engines;
- vector databases;
- ranking models;
- user-interface patterns;
- future limits of intellectual activity.

These may be important, but they belong outside the Core.

Some belong to Projection.

Some belong to authoring methods.

Some belong to retrieval architecture.

Some belong to implementation.

Some belong to future intellectual activities not yet anticipated.

## 6.4 Core and Projection Boundary

Projection is not part of the Core in the strict sense.

Projection is a reuse strategy applied to original sources under particular purposes, situations, users, organizations, or future search needs.

The Core must allow Projection to vary.

Different projections may legitimately produce different entities, occurrences, sections, and retrieval structures from the same original source.

This is not an error.

It is one of the central design assumptions of IdeaMark.

The Core should therefore never encode assumptions that belong to individual projections.

## 6.5 Goals Remain Outside the Core

IdeaMark intentionally does not define goals.

Goals belong to situations, projections, users, organizations, AI systems, and future intellectual activities.

A system may use IdeaMark for planning, execution, evaluation, reflection, education, policy design, field operation, software architecture, investment analysis, disaster prevention, or other activities.

The Core should not decide which of these is primary.

Instead, it should preserve reusable structures that allow future interpreters to discover, compare, and reconstruct possible paths under their own goals.

## 6.6 Workflows Remain Outside the Core

IdeaMark does not prescribe a single authoring, retrieval, validation, or interpretation workflow.

An IdeaMark document may be authored by humans, AI systems, or both.

It may be used by CLI tools, web applications, databases, retrieval systems, LLM agents, or human readers.

It may support lightweight search, deep analysis, batch processing, interactive dialogue, or field operation.

The Core should define what must remain structurally meaningful across these workflows, not how each workflow must be performed.

## 6.7 Implementations Remain Outside the Core

IdeaMark Core is not a database design, search engine design, or application architecture.

It should remain compatible with many implementation choices.

For example, an implementation may use relational databases, document databases, graph databases, vector search, file-based storage, or hybrid systems.

The Core should not prefer one storage or retrieval implementation unless doing so is necessary for interoperability.

## 6.8 Engineering Minimalism

IdeaMark Core follows engineering minimalism.

The Core intentionally defines only the minimum engineering boundary required for reusable intellectual structures.

Anything that unnecessarily constrains future intellectual activity should remain outside the Core whenever possible.

This minimalism is not a lack of ambition.

It is the condition that allows ambitious future projections, authoring methods, retrieval systems, AI systems, and social practices to evolve.

## 6.9 Openness to Future Intellectual Activity

IdeaMark intentionally avoids defining the limits of intellectual activity.

Future intellectual activities may be performed by humans, AI systems, organizations, instruments, communities, or combinations not yet anticipated.

The Core should therefore define reusable structural mechanisms rather than a closed taxonomy of activities.

By leaving meanings, goals, vocabularies, coordinate systems, and activity boundaries open, IdeaMark remains available to future domains and future forms of human-AI co-evolution.

## Design Rationale

Design Scope is not merely about what the Core includes.

It is about how the Core protects future freedom.

If IdeaMark attempted to define meaning, cognition, universal vocabularies, universal coordinates, or universal goals, it would become tied to the interpretive assumptions of a particular time, domain, community, or AI system.

By defining only reusable structural access to original sources, IdeaMark remains open to future projections, AI systems, media forms, domains, workflows, and intellectual practices.

The purpose of the Core is not to prescribe how intellectual activities should be performed.

It is to define a universal environment in which many different intellectual activities can be represented, reused, reconstructed, and continuously evolved.

# IdeaMark Examples

This section contains example IdeaMark documents and example transformations.

The purpose of these examples is not to demonstrate a single correct interpretation. Their purpose is to show how reusable intellectual traces can be represented and reused.

## Planned Example Categories

### Minimal Examples

Small examples focused on individual concepts.

Examples:

- single Entity
- Entity + Occurrence
- Entity + Occurrence + Section
- simple Relations
- simple Perspectives

### Transformation Examples

Examples showing how IdeaMark structures can be derived from source material.

Examples:

- conversation -> IdeaMark
- report -> IdeaMark
- observation log -> IdeaMark
- research note -> IdeaMark

### Reuse Examples

Examples showing how the same IdeaMark structure may support different projections.

Examples:

- discussion -> planning
- observation -> policy
- evidence -> argument
- business discussion -> TPCG planning

### Cross-Domain Examples

Examples showing reuse across different domains.

Examples:

- research
- business
- policy
- education
- knowledge synthesis

## Design Principle

Examples should demonstrate that IdeaMark preserves reusable traces of intellectual activity rather than preserving a single fixed interpretation.
